В этой функции есть элементы двух типов:

1. **могут вызывать нормальное ветвление выполнения**
2. **могут приводить к генерации исключения**

Разберём по строкам.

```cpp
void test ( Person const & person )
{
    std::cout << " test : " << person.name() << '\n';
    if ( person.grade() == 10 || person.salary() > 1'000'000)
    {
        save ( Status::success , person.id() ) ;
    }
    else
    {
        save ( Status::failure , person.id() ) ;
    }
}
```

## Что создаёт ветвление

Главный элемент ветвления — это условный оператор:

```cpp
if ( person.grade() == 10 || person.salary() > 1'000'000 )
```

Здесь есть сразу несколько точек, влияющих на путь выполнения:

* `person.grade() == 10`
  даёт булев результат: истина или ложь.

* `person.salary() > 1'000'000`
  тоже даёт булев результат: истина или ложь.

* оператор `||`
  реализует **короткое замыкание**:

  * если `person.grade() == 10` истинно, правая часть уже **не вычисляется**;
  * если ложно, тогда вычисляется `person.salary() > 1'000'000`.

Из-за этого возможны разные нормальные пути выполнения:

* переход в ветку

  ```cpp
  save(Status::success, person.id());
  ```
* переход в ветку

  ```cpp
  save(Status::failure, person.id());
  ```

## Что может генерировать исключения

Поскольку `Person` — пользовательский класс, а `save` — внешняя функция, практически каждый вызов пользовательского кода потенциально может бросить исключение.

### 1. `person.name()`

Используется здесь:

```cpp
std::cout << " test : " << person.name() << '\n';
```

Возможные причины исключения:

* метод `name()` сам может бросить исключение;
* возвращаемое значение может требовать копирования/выделения памяти;
* при выводе в поток `std::cout` возможны исключения потока, если для него включены исключения через `exceptions()`.

### 2. Операция вывода в `std::cout`

Вся строка

```cpp
std::cout << " test : " << person.name() << '\n';
```

может привести к исключению из-за:

* ошибки вывода;
* исключения внутри перегруженного `operator<<` для типа, который возвращает `person.name()`;
* исключения самого потока `std::ostream`, если они разрешены.

### 3. `person.grade()`

В условии:

```cpp
person.grade() == 10
```

метод `grade()` может бросить исключение.

### 4. `person.salary()`

В условии:

```cpp
person.salary() > 1'000'000
```

метод `salary()` тоже может бросить исключение.

### 5. `person.id()`

Используется в обеих ветках:

```cpp
person.id()
```

Этот метод тоже может бросить исключение.

### 6. `save(...)`

Оба вызова:

```cpp
save(Status::success, person.id());
save(Status::failure, person.id());
```

могут бросать исключения, например:

* ошибка записи;
* ошибка доступа к файлу, БД, сети;
* внутренняя логическая ошибка функции `save`.

## Важная деталь про `||`

В выражении

```cpp
person.grade() == 10 || person.salary() > 1'000'000
```

из-за короткого замыкания:

* если `person.grade() == 10` истинно, `person.salary()` **не вызывается**;
* значит, в этом случае исключение из `salary()` возникнуть не может.

Это тоже влияет на путь выполнения.

## Итог

### Элементы, ведущие к нормальному ветвлению:

* `if`
* сравнение `person.grade() == 10`
* сравнение `person.salary() > 1'000'000`
* логический оператор `||`
* выбор между ветками `then` и `else`

### Элементы, которые могут генерировать исключение:

* `person.name()`
* вывод в `std::cout`
* `person.grade()`
* `person.salary()`
* `person.id()`
* `save(...)`

### Возможные пути выполнения:

* успешный вывод + ветка `success`
* успешный вывод + ветка `failure`
* исключение при выводе имени
* исключение при получении `grade()`
* исключение при получении `salary()` (только если левая часть `||` ложна)
* исключение при получении `id()`
* исключение внутри `save(...)`
